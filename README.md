Odpowiedzi na pytania:
#1 Kiedy git wykona fast-forward, a kiedy powstaje merge commit?
Gdy po przejsciu na nowa galaz, na maine nie doszlo do zadnych 
commitow wtedy mozna po prostu przepiac pliki na glowna galaz, merge commit powstaje
gdy historia np. main juz sie zmienila
#2 Czym w praktyce rozni sie merge od rebase?
Merge - zachowuje prawdziwa historie projektu
Rebase - przepisuje historie i sprawia ze jest ona liniowa
#3 W jaki sposób został rozwiązany konflikt w Twoim repozytorium?
Wytwarzajac konflikt zmienilem w dwoch galeziach nazwy zmiennej tej samej metody, aby to rozwiazac
musialem wybrac ktora nazwe chce zostawic.
